Bonnes pratiques de code :
========================

- Vérifier et prêter attention à  l’indentation
- Commenter le plus possible le code
- Utiliser un maximum les composants pour diviser au plus le code et pouvoir les réutiliser sans dupliquer le code
- Utiliser les camel case 

Gestion du git :
==============

- faire les commits sur sa propre branche, cela permet d'avoir aucun bug ni problème de merge sur la branche master
- quand une fonctionnalité est au point ajouter son travail à la branche master : 
	1. git pull origin master
	2. git push origin nomDeLaBranche
	3. git checkout master
	4. git pull origin nomDeLaBranche
	5. git push origin master
	6. git checkout nomDeLaBranche
